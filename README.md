# Shotlytics — AI Basketball Shooting Form Analyzer

An AI-powered web app that analyzes basketball shooting form from a single video upload. Automatically detects the load position, release point and follow through, calculates biomechanical measurements at each stage, and generates a personalized coaching report using an LLM.

---

## Demo

Upload a basketball shooting video → get instant analysis of your shot form with annotated frames and AI coaching.

---

## What it analyzes

**Load Position** — maximum knee bend before the shot
- Left and right knee angles
- Elbow angle at load
- Hip squareness
- Body balance

**Release Point** — moment the ball leaves the hand
- Elbow angle
- Knee angle
- Wrist position relative to shoulder
- Elbow alignment (stacked vs flared)

**Follow Through** — frames after release
- Elbow extension
- Body balance after release

**AI Coaching Report** — personalized coaching feedback generated by Llama 3.3 70B via Groq

## Tech Stack

| Layer | Technology |
|-------|-----------|
| Ball Detection | YOLOv8x |
| Ball Tracking | OpenCV CSRT Tracker |
| Pose Estimation | MediaPipe Pose |
| Fallback Detection | HSV Color Filtering |
| AI Coaching | Groq API (Llama 3.3 70B) |
| Backend | Flask + SSE streaming |
| Frontend | HTML, CSS, JavaScript |

---

## How it works

1. **Pass 1** — YOLO scans frames to locate the basketball. Falls back to HSV orange color detection if YOLO fails.
2. **Pass 2** — CSRT tracker follows the ball frame by frame. MediaPipe detects body pose simultaneously. Load position tracked as the frame with minimum knee angle. Release detected when ball-to-wrist distance suddenly spikes.
3. **Analysis** — Three frames analyzed: load, release, follow through. Joint angles calculated using dot product formula.
4. **Coaching** — All measurements sent to Groq API. Llama 3.3 70B generates a concise personalized coaching report.
5. **Streaming** — Real-time progress updates sent to browser via Server Sent Events (SSE) as each step completes.

## Project Structure

- **stanceanalyzer.py** — main pipeline, ball detection and tracking, orchestrates the full analysis
- **utils.py** — shared helper functions for angles, drawing and landmark extraction
- **load_analysis.py** — analyzes the load position frame
- **release_analysis.py** — analyzes the release frame
- **followthrough_analysis.py** — analyzes the follow through frame
- **feedback.py** — builds measurements summary and calls Groq API for coaching report
- **app.py** — Flask backend with SSE streaming for real-time progress updates
- **templates/index.html** — frontend UI
- **static/style.css** — styling
- **static/script.js** — frontend logic, SSE event handling, results rendering

## Setup
 
1. Clone the repo
```
git clone https://github.com/yukti-0411/basketball-shooting-form-analyzer
cd basketball-shooting-form-analyzer
```
 
2. Install dependencies
```
pip install flask opencv-python mediapipe ultralytics numpy requests
```
 
3. Set Groq API key
```
$env:GROQ_API_KEY="your_key_here"
```
 
4. Run
```
python app.py
```
 
5. Open browser at `http://127.0.0.1:5000`

## Tips for best results
- Ball and full body must be clearly visible in the frame
- Video should contain only the shooting motion — no dribbling
- One person in focus, good lighting
- For fast shooters use 0.5x or 0.25x speed option in the UI


## Current limitations:
- Works best at a 45 degree angle to the player — pure side view misses elbow alignment, pure front view makes knee and elbow angles less accurate
- Ball detection may fail in poor lighting or if the ball is occluded by hands
- Wrist snap not analyzed — MediaPipe pose model does not provide hand keypoints for reliable detection
- Designed for single-person videos only — multiple people in frame may confuse pose detection
- Load position detected as minimum knee angle — may not always correspond to the true load position for shooters with unconventional form
- Release detection based on ball-to-wrist distance spike — may occasionally misfire on pump fakes or pass attempts if video contains them
- Analysis quality depends on video resolution and frame rate — very low quality videos may produce inaccurate measurements
